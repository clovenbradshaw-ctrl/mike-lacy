# EO

A framework for tracking how things change.

Most databases store what something *is*. EO stores what *happened* to it — nine operators that record every transformation a piece of information undergoes, so that any state can be traced back through every decision that produced it.

It's an ontology built for complete auditability — designed so that systems can't hide what they've done.

- Started as philosophy at St. John's College
- Became a data architecture
- Asks the same question the journalism asks: *who changed what, when, and who approved it?*
